React hooks

Why are React Hooks needed?

  • simplifying expected behavior
  • increasing extensibility
  • improving performance

hooks wich we're talking about:

useState, useRef, useEffect, useMemo

useState(initialState)

Returns a stateful value, and a function to update it.

We don't change the state of the component directly, we call the set function, where we pass the value of the component. 

              
                const [state, setState] = useState(initialState);

useRef manages the DOM tree directly and returns a mutable ref object whose .current property is initialized to the passed argument (initialValue). 

                
                  const someElement = useRef(initialValue);

useEffect(callback, deps)

Tracks the stages of the life cycle.

life cycle

  • mount (creating and mounting a component in a DOM tree)
  • update (see and use a component)
  • unmount (when the component is not needed it is removed)
  • performs the first display during the mount stage
  • Keeps track of the change in the state of the array elements and calls the code when the change occurs - update stage
  • For example when we go to another page and React removes a component - unmount stage

useMemo(callback, deps)

Callback must return the result of some calculations (for example, a sorted array)

              
                useMemo(() => {
                  return [...users].sort(...)
                }, [selectedSort, users])

callback will only be called if any of the dependencies change their value
